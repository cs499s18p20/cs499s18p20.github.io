Project Requirements

Essential Requirements

  1. Successful reproduction of an object recognition tutorial. We will be following this guide that was shared with us by the customer.
  2. Creation of a library of annotated simple card images. This library of images will be used throughout the project to train the model. As we work to improve the model to recognize more advanced patterns, this library will be expanded as necessary for training. Annotation will likely be done using this tool.
  3. Addition of simple "card objects" to the model trained in the object recognition tutorial. This will require adding our library of annotated card images to the training data used in the tutorial. The goal here is to have the neural net successfully recognize and identify images of cards. It will not be able to identify suit or value at this point.
  4. Removal of the original dataset from the tutorial. This will leave our model with only the card images to train from and will be a turning point in that our project will now be quite disjoint from the original tutorial we started with.
  5. Addition of the ability to differentiate between suits of cards. This may or may not cover all 4 suits depending on the time remaining once we reach this point in the project. The original goal will be to differentiate between two different suits.
  6. Addition of the ability to differentiate between numbers of cards. This will not likely cover all 13 different values since that would require the creation of a massive library of annotated images. It should be able to differentiate between a specifically chosen group of card values.

Stretch Goals

  1. Ability to identify all 52 cards. This is an extension on the last two essential requirements and just expands these to cover all cards in the deck.
  2. Ability to identify overlapping and truncated card images. This would require the creation of a new dataset of images that would allow us to train the model to recognize cards even in these non-ideal forms.
  3. Mobile or web interface to allow remote image identification. This would allow a user to take an image on their phone and submit it to the neural net and receive immediate feedback about what cards are in the image.

Schedule and Milestones

  • Week 1: Finish Tutorial (Requirement 1) and successfully run using Google Cloud Platform.
  • Week 2: Finish creation of simple dataset (Requirement 2) and train the existing model to see if it is able to properly recognize the images of cards (Requirement 3).
  • Week 3: Refine the simple dataset and continue working with the model until it is able to successfully recognize all simple card images in the dataset (Requirement 3).
  • Week 4: Remove the original training data from the model leaving only the card images (Requirement 4).
  • Week 5: Continue to refine existing model, potentially adding more images to the training sets until it is correctly able to identify all cards in the test dataset.
  • Week 6: Begin training and testing with the next level dataset to train differentiation between suits (Requirement 5).
  • Week 7: Continue work from week 6 (Requirement 5).
  • Week 8: Begin training and testing with the next level dataset to train differentiation between card values (Requirement 6).
  • Week 9: Continue work from week 8 (Requirement 6) and add more suits and values if possible (Stretch 1).
  • Week 10: Attempt to train overlapping, obstructed, and truncated card image recognition (Stretch 2).

Platforms, Tools, and Languages

  • TensorFlow: Google's open-source machine learning framework.
  • labelImg: An open-source image annotation tool.
  • Google Cloud Platform: Google's suite of cloud based computing services.
  • Python: The preferred programming language for working with TensorFlow.
  • Internal tools: It is very likely that we will find it helpful to build some of our own tools (for example, a script to rotate images in order to create more test images from a single picture). These will be added as we develop them. Stretch Goal 3 may be another tool that we find useful.
