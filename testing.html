Test Plan

This project involves several pieces that require testing: the card identification model, the back-end server, front-end application, whether they are cross-platform capable, various hardware and software combinations, and whether the interface is user friendly and bug free. This is the team's plan to resolve most, if not all, issues related to that.

  • Unit Testing
    1. Model
      1. Test against an image with no cards in it. Does it return “None” or something reasonable like that?
      2. Test every individual card, i.e. 9-A of all four suits. Does it identify them correctly?
      3. Test an image of a Euchre Hand. Does it identify every truncated image correctly?
      4. Test an image with multiple Euchre hands. Does it identify every card image correctly?
      5. Test against various Euchre hands and combinations of known card types. Does it identify every card correctly?
      6. Test against cards that are not included in a Euchre deck, like 8 of Hearts. Does it return “None” or something reasonable like that?
      7. Test against an image with every Euchre card in it. Does it recognize all of the cards?
      8. Test against non-standard card images. Does it still recognize the card?
    2. Server
      1. Send an image to the server. Does it respond? Does it send back an image?
      2. Send a non-image file to the server. Does it respond? Does it handle invalid input appropriately?
      3. Send an image to the server, wait for its response, send corrections to the server. Does it make the corrections and send them back to the user?
      4. Send a NULL input to the server. Does it handle it appropriately/gracefully?
    3. Web Application
      1. Attempt to send an empty input to the server. Does it disallow an empty input? Does it require a file before sending to the server?
      2. Attempt to send a file without specifying the trump. Does it not allow the user to do so?
      3. Attempt to send an invalid file type, like .xml file to the server. Does it not allow the user to do so?
      4. Is the web page viewable to users on the www, i.e. can one access it from outside the local machine?
      5. Is it viewable/readable on a mobile device?
  • Integration/Functional Testing
    1. Go to web page. Is it available to users, i.e. can one access it outside of the local machine?
    2. Send image and trump suit to server. Does it respond? Does it reply with the labelled and the cards in the image?
  • System Testing
    1. Windows 8.1
      1. Google Chrome
      2. Internet Explorer (IE)
      3. Mozilla Firefox
    2. Windows 10
      1. Google Chrome
      2. Internet Explorer (IE)
      3. Mozilla Firefox
    3. Mac OSX
      1. Google Chrome
      2. Internet Explorer (IE)
      3. Mozilla Firefox
    4. Ubuntu
      1. Google Chrome
      2. Internet Explorer (IE)
      3. Mozilla Firefox
  • User/Acceptance Testing

    Since different group members will have more experience developing certain parts of the application than others, we will have those less experienced with the web application for testing that end of it and providing feedback to the most experienced members of each component.
